## Four Go/No-Go Gates

### Gate 1 — Unit 03: NUT-11 P2PK CLI Round-Trip
Stop if the script does not print `REDEEM OK`, `DOUBLE-SPEND DETECTED`, and `DLEQ OK`.
**Downstream impact if skipped:** Every unit from 08 onward assumes mint P2PK support. Silent double-spend acceptance breaks the entire payment security model.

### Gate 2 — Unit 06: ICE Connection State "connected"
Stop if two browser tabs do not reach `iceConnectionState: connected` on localhost.
**Downstream impact if skipped:** Units 07, 10, 11, 12, 13, 14, 17, 19 all require a working RTCPeerConnection.

### Gate 3 — Unit 10: Manual Token Transfer + Double-Spend Rejection
Stop if viewer cannot send a token that tutor redeems and acks, OR if double-spend is not rejected with `reason: "double_spend"`.
**Downstream impact if skipped:** Unit 11 automates this flow. If the manual flow is broken, the scheduler retry logic (which re-sends on timeout) is indistinguishable from a double-spend attack.

### Gate 4 — Unit 11: Payment Scheduler with Budget Exhaustion
Stop if scheduler unit tests fail (wrong chunk count, wrong exhaustion timing).
**Downstream impact if skipped:** Session UI (Unit 12) depends on scheduler events. Budget guard is a safety mechanism — skipping verification could overcharge or undercharge viewers.
